# DEVELOPMENT_PLAN

## Цель
Создать стабильное и гибкое решение для Luckfox Camera RTSP с удобным управлением через MQTT, поддержкой Frigate/HA и эволюционирующей функцией детекции на камере.

## 0. [ВАЖНО] Перейти на VI→VPSS→VENC (основной видео-пайплайн)
- Цель: уйти от ручной схемы VI -> RGA copy/resize -> VENC и получить максимально аппаратный zero-copy путь.
- Ожидаемый эффект: ниже CPU-нагрузка, меньше джиттер main stream, выше стабильность при длительном аптайме.

### Что нужно сделать
- Добавить инициализацию VPSS группы/каналов для двух веток:
  - VPSS Ch0: passthrough для main stream (без масштабирования).
  - VPSS Ch1: масштабирование для sub stream (640x360 по умолчанию).
- Выполнить bind модулей в RK MPI:
  - VI ch0 -> VPSS grp0.
  - VPSS ch0 -> VENC ch0 (main).
  - VPSS ch1 -> VENC ch1 (sub).
- Перевести mirror/flip в VPSS (или в ISP/sensor где поддерживается) и удалить дублирование transform через RGA в main loop.
- Упростить main loop до чтения VENC stream и RTSP tx (без ручной копии кадров и без ручного resize).
- Сохранить runtime-управление fps/bitrate через MQTT для main/sub и корректный пересчёт GOP.
- Добавить безопасный fallback:
  - если bind/VPSS не поднялся, запускать текущий RGA-пайплайн как запасной режим (через конфиг/флаг).

### Что обязательно учесть
- Порядок старта/остановки модулей и bind/unbind (иначе утечки буферов или зависание при рестарте сервиса).
- Параметры буферизации и depth у VI/VPSS/VENC:
  - не допустить backpressure, когда sub stream тормозит main stream.
- Таймстемпы и time-ref:
  - PTS должны быть монотонными и согласованными для стабильного RTSP playback в Frigate/FFmpeg.
- Совместимость сенсоров MIS5001/SC3336:
  - mirror/flip и day/night не должны ломать ориентацию или экспозицию после миграции.
- Диагностика и наблюдаемость:
  - добавить явные метрики/логи по pipeline mode, дропам кадров, stall-событиям и времени восстановления.

### Что проверить после внедрения
- Функционально:
  - RTSP main (/live/0) и sub (/live/1) поднимаются и работают параллельно.
  - MQTT-команды fps/bitrate применяются без перезапуска процесса.
  - mirror/flip и day/night работают одинаково на обоих сенсорах.
- Производительность:
  - CPU usage ниже текущего baseline на типичном профиле (2K main + 640x360 sub).
  - Снижение jitter/frame-time spikes в main stream.
  - Нет роста dropped frames при одновременных клиентах Frigate + VLC.
- Надёжность:
  - soak-test 24-72 часа без stall и без деградации памяти.
  - watchdog/restart path корректно восстанавливает стримы.
  - корректное завершение сервиса без зависших bind и без повреждения настроек.

## 1. Тестирование микрофонов
- проверить совместимость с USB, I2C, SPI и аналоговыми микрофонами (Pico Ultra + другие платы).
- измерить качество звука, задержку, SNR, артефакты и устойчивость в реальном случае.
- выбрать оптимальный аудио-стриминг и параметры AEC/AGC/HPF для финальной сборки.
- прописать результат в документации и в техпаспорте `audio_mpi`.

## 2. Корпус на 3D-принтере
- разработать дизайн с учётом охлаждения, антенн, разъёмов и управления кабелями.
- обеспечить герметизацию/защиту от пыли, и при необходимости конденсата.
- сохранить STL/описание в `deploy/case/`, добавить сборочные инструкции.

## 3. RTSP авторизация
- внедрить username/password для потоков RTSP (main/sub).
- поддержать digest/basic, совместимые с FFmpeg/Frigate.
- добавить MQTT-опции `rtsp_user`, `rtsp_pass`, `rtsp_auth_mode`.
- в документации описать настройку и случаи fallback.

## 4. Корректировка MQTT-данных
- провести ревизию `state` и `telemetry`, убрать устаревшие и лишние поля.
- добавить реально важные поля:
  - `stream_bandwidth_actual`
  - `substream_fps_effective`
  - `cpu_temp`, `soc_temp`, `voltage`, `power_w`
  - `video_dropped_frames`, `packet_loss` (если возможно)
  - `audio_status`, `mic_level`, `audio_error`
- сделать режимы `telemetry_mode=light/detailed`.
- зафиксировать контракт топиков в `MQTT_CONTRACT.md`.

## 5. NPU + детекция на камере
- базовый MVP: локальная детекция (person/motion) в `substream` через RKNN.
- собрать pipeline: capture -> postprocess -> inference -> MQTT alert.
- обеспечить низкий TDP: режимы `detection_interval`, `skip_frames`, `low_power`.
- добавить флаг `edge_detection_enabled` и настройки модели.
- продолжить интеграцию Frigate через MQTT-события и RTSP detect stream.

## 6. Дополнительные идеи
- OTA обновления + резервный раздел для безопасного отката.
- локальный мини web UI (HTTPS) для контроля параметров, debug и live preview.
- режим энергосбережения: `sleep`/`wake-on-motion`, fps downscale и bitrate downscale.
- `camera_rtsp.json` v2: версия, checksum, миграция настроек при апгрейде.
- CI проверка: unit + integration + lint + статический анализ.

## Этапы и приоритеты
1. критично: переход на VI→VPSS→VENC и проверка стабильности/производительности.
2. безопасность: RTSP auth, MQTT контракт, стабильность хранения настроек.
3. надёжность: мониторинг/телеметрия, graceful shutdown, FSM ошибок.
4. UX: корпус и удобная документация.
5. продвинутая фича: on-device NPU детекция.
